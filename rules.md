# Rules

## Notes page rules

- Keep `revision.md` as the single main notes page for revision content.
- Do not split revision content across `TODO.md` and `revision.md` again unless explicitly asked.
- Do not use absolute filesystem markdown links like `C:/...` inside note pages.
- Prefer plain filenames like `revision.md`, `TODO.md`, `rules.md` inside notes when referring to local files.
- Do not add links that open in Chrome when a simple plain-text filename is enough.
- When making revision notes, prefer one-page consolidation with a small index at the top.
- Keep notes interview-focused: deep explanation, short takeaway, keywords when useful.

## Current structure

- `revision.md` = main combined revision page
- `TODO.md` = short pointer page only
- `rules.md` = standing note rules
